# Contributing to Open Comet

We're excited that you're interested in contributing! Open Comet is an open-source project, and we welcome contributions from everyone.

## 🚀 How to Get Started

1.  **Fork the Repository**: Create your own copy of the project to work on.
2.  **Clone Locally**: Download your fork to your machine.
3.  **Create a Branch**: Use a descriptive name like `feature/new-skill` or `fix/auth-warning`.
4.  **Implement Your Changes**: Follow the project's coding style (standard JavaScript, JSDoc for public APIs).
5.  **Test Your Changes**: Ensure your changes don't break existing functionality.
6.  **Submit a PR**: Provide a clear description of what your changes do and why they're needed.

## 🛠️ Development Setup

1.  Load the extension in Chrome via `chrome://extensions` using "Load unpacked".
2.  Make your changes in the `src/` directory.
3.  Refresh the extension in Chrome to see your changes.

## 📋 Code Style Guidelines

-   **JSDoc**: Add JSDoc comments to all new functions.
-   **Naming**: Use `camelCase` for variables and functions.
-   **No dead code**: Remove all `console.log` or `debugger` statements before submitting.
-   **Comments**: Use `// TODO:` or `// FIXME:` to signal areas for future improvement.

## 💬 Reporting Issues

-   Use the GitHub Issue tracker to report bugs or suggest features.
-   Provide as much detail as possible, including screenshots and steps to reproduce.
